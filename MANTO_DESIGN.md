# Manto — 设计风格指南

> Web3 暗金美学 · 神圣触感 · 五语适配
> 2026-06-26

---

## 一、设计理念

**Manto 看起来不像教堂。它像你凌晨三点还在刷的手机里，泛着暗金微光的那个角落。**

三条核心设计原则：

1. **触感优先** — 用户不是在「操作」，是在「触摸神圣」。不用按钮的地方绝不用按钮。手指按住比点击更接近祈祷。
2. **暗金不刺眼** — 深黑底 + 暗金呼吸光。暗室中使用不刺眼，阳光下仍可辨。web3 用户在深夜打开的比例远超日间。
3. **神不是 logo** — 每尊神像一个角色，有自己的颜色、性格、说话方式。Aparecida 不是 Guadalupe 换皮。观音不是圣母复制品。

## 五、动画规范

```
微交互:     200-300ms ease-out
             用于按钮 hover、tooltip 出现、toast 弹入

页面过渡:    300-400ms ease-in-out
             用于页面切换、模块展开

牌面翻转:    400ms cubic-bezier(0.4, 0, 0.2, 1)
             用于塔罗牌 3D 翻转

粒子动画:    持续循环，帧率 ≥30fps
             用于星空背景、参拜粒子、牌背粒子

呼吸光:     周期 3-4s，ease-in-out 循环
             用于牌背、神像光环、按钮辉光

页面滚动:    smooth scroll behavior
             用于列表和历史记录
```

**参拜动画详细曲线：**

```
0-3s 段    ease-out
           halo: radius 0→30px
           particles: count 0→15, speed slow
           brightness: 1.0→1.15

3-7s 段    linear
           halo: radius 30→80px
           particles: count 15→40, speed accelerating
           vignette: opacity 0→0.3
           deity-edge: golden glow onset

7-10s 段   ease-in-out
           halo: radius 80→120px
           particles: count 40→80, curving
           mandala: opacity 0→0.6 (玫瑰窗/莲花)
           screen-tint: warm gold 0→0.08

10s+       ease-out
           particles: 90% fall like golden rain
           deity: scale 1.0→1.03
           flash: white overlay 0→0.15→0 (200ms)
           → transition to end screen
```

**开关控制：** 参拜动画可被系统「减少动态效果」设置影响。当 `prefers-reduced-motion` 为 true 时，禁用粒子动画，仅保留静态光环呼吸。

## 八、神圣日视觉变体

某些日期触发特殊的全局视觉：

### 守护神节日

当日如果是用户守护神的节日：
- 首页牌背呼吸光颜色从暗金临时切换为该神的主色
- 参拜页 halo 颜色强化为该神辅色
- 今日指引文案末尾自动追加一句节日祝福

```
Guadalupe 日 (12/12):
  全站金色替换为 Guadalupe 玫瑰红 #8B2E3A
  牌背纹样中暗金星芒替换为玫瑰纹样
  粒子颜色从金色改为玫瑰金
```

### 满月/新月

```
满月：
  粒子密度翻倍
  牌背呼吸光周期从 3s 缩短至 2s
  今日能量底部显示 🌕 满月 · 能量双倍

新月：
  粒子密度减半
  牌背呼吸光更暗
  首页出现 🌑 新月 · 种下心愿 入口
```

- 神圣日变体不影响品牌一致性
- 所有变体在日期结束后自动恢复

## 九、暗色模式唯一

Manto 只有暗色模式。不提供亮色切换。

原因：
1. 品牌性格——暗金美学是 Manto 的视觉身份
2. 使用场景——凌晨打开的用户占比远高于其他 App
3. 宗教心理——暗光环境中的神圣感比明亮环境强
4. 维护成本——单模式让组件库简单一倍，移动端包体更小

## 十、无障碍

虽然只有暗色模式，仍需确保可访问：

- 正文文字与背景对比度 ≥ 4.5:1（#E8E0D5 对 #0D0D0D = 12.5:1 ✓）
- 按钮文字与背景对比度 ≥ 3:1
- 参拜动画受 `prefers-reduced-motion` 控制
- 语音输入为占卜问题作为替代输入方式
- 图像（神像、牌面）提供 alt 文本

## 十一、响应式断点

```
mobile:     < 640px    单列布局，全宽卡片
tablet:     640-1024px 双列网格（水晶商城）
desktop:    > 1024px   居中内容 max-width 800px，侧边栏
```

移动端优先设计，桌面端为移动端的居中放大版 + 键盘快捷键。

### 移动端特殊适配

- 底部导航需要扣除 `safe-area-inset-bottom`
- 触控区域最小 44×44px（Apple HIG 标准）
- 翻牌手势考虑拇指热区（屏幕下半部分优先放置 CTA）
- 参拜触控区域放在屏幕中央偏下（拇指最舒适位置）
- 输入框弹出时页面自动上推，不遮挡输入区域

## 十二、插图与图标

### 神像规格

每尊神像为定制 SVG，非照片、非 AI 生成：

- 风格：扁平化圣像画风，线条流畅，色调基于该神的主辅色
- 表情：宁静、温和。不夸张、不戏剧化
- 不写实——写实神像在不同文化中可能引发「偶像崇拜」争议
- 8 尊核心神像为一次性委托，后续众筹上线神祇逐步补充

### 塔罗牌面

- 78 张韦特塔罗牌的现代化 SVG 重绘
- 保持韦特体系的可辨识度（每张牌的关键符号保留）
- 色调统一为暗金体系，不使用原版韦特的饱和原色
- 牌面不追求完全还原 Rider-Waite——追求「认出这张牌 + 融入 Manto 氛围」

### 图标

- 系统图标使用 Lucide React（当前项目已集成）
- 图标颜色统一使用 #E8E0D5 或 #C9954A，不引入新颜色
- 底部导航图标尺寸 22px
- 列表/卡片内图标尺寸 16-18px

## 十三、文案基调

### 中文

亲切、有点古风但不拗口。像一位懂现代生活的道士在跟你说话。

可以用："翻一张牌，看看今天怎么走"
不可以用："宇宙的能量正在向您汇聚"

### 葡语/西语

温暖直接，像 WhatsApp 上那位信教但不说教的阿姨。

PT: "Vire uma carta. Veja como o dia caminha."
不: "O universo está convergindo suas energias para você."

### 英语

Discord-friend tone. 亲和、偶尔有点嘲讽但善意。

EN: "Flip a card. See what's up."
不: "The cosmic energies are aligning in your favor."

### 五个通用规则

1. 不用感叹号连接成串（"太棒了！！！"）
2. 不用敬语拉开距离（葡语的 você 不用 o senhor/a senhora）
3. 不说绝对话（"一定会成功" → "牌面显示这方向不错"）
4. 承认不确定性（"牌也不一定对——但看看呗"）
5. 不装神弄鬼——用户知道对面是 AI，文案别假装是真人塔罗师
